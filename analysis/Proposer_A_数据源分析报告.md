## 1. ESPN 数据源现状与局限

### 当前已使用的 ESPN 字段

| 字段 | 用途 | 可靠性 |
|------|------|--------|
| 比赛日程 (date/kickoff) | 赛程安排 | ✅ 稳定 |
| 比分 (score) | 回填结果、校准 | ✅ 稳定 |
| 球队 form (W/D/L 5场) | form_to_score | ✅ 稳定 |
| 锦标赛 record (W-D-L) | record_to_score | ✅ 稳定 |
| ML 赔率 (details) | 隐含概率 | ⚠️ 经常为空 |
| 平局 ML (drawOdds.moneyLine) | 三向去水 | ⚠️ 有时缺失 |
| 亚盘 open/close (pointSpread) | spread_movement | ✅ 最稳定字段 |
| 大小球 (total over/under) | 大小球方向 | ✅ 稳定 |

### ESPN 关键局限

| 分类 | 缺失数据 | 对预测的影响 |
|------|----------|-------------|
| **高级统计** | xG (期望进球)、xA (期望助攻)、射门、控球率、传球 | **严重** — 当前 Poisson λ 纯靠赔率+form 推算，与实际比赛质量脱节 |
| **球员级数据** | 阵容、伤病、停赛、首发 | **严重** — 关键球员缺阵无法反映在预测中 |
| **多源赔率** | 仅 DraftKings 一家 | **中等** — 单一博彩公司赔率可能有偏差，无法做市场共识 |
| **赔率历史** | 只有 open/close，无盘中走势 | **中等** — 无法捕捉盘中资金流向 |
| **历史数据** | 仅当前锦标赛窗口 | **中等** — 校准样本不足（当前仅 66 场） |
| **天气数据** | 无场地天气信息 | **低-中** — 雨/雪/高温影响比赛风格 |
| **H2H 历史** | 无两队历史交锋记录 | **低-中** — 对某些对决有意义 |
| **场地/裁判** | 无场地、裁判信息 | **低** |
| **API 稳定性** | 非官方隐藏 API，可随时变更 | **高** — 2026-06 已多次调整 |
| **联赛覆盖** | 仅 FIFA 世界杯联赛，非五大联赛 | **未来扩展时严重** |

### 当前数据流中的关键缺口

> 当前模型用 `home_true_prob * 0.40 + form * 0.20 + record * 0.15 + spread_movement * 0.25` 推算 λ，但：
> 1. **无 xG 校准** — 实际进球质量 vs 期望进球的差异无法捕捉
> 2. **无伤病信息** — 梅西/姆巴佩缺阵这种降维打击无法体现
> 3. **单博彩公司** — DraftKings 的 7% 水钱固定，无法跨公司比较

## 3. 重点数据源深度评估

### 3.1 赔率数据源

#### 当前：ESPN (DraftKings 独家)
- **现状**：单家博彩公司，ML 字段经常为空，只有 open/close 两帧
- **局限**：$30/月 的 The Odds API 就能拿 10+ 家博彩公司数据

#### 推荐：The Odds API (https://the-odds-api.com)
- **免费版**：500 积分/月 ≈ 16 次请求/天 → 够世界杯单日赛程
- **$30/月 版**：20,000 积分/月 + 历史赔率回溯
- **覆盖**：DraftKings, FanDuel, BetMGM, Pinnacle, 1xBet 等 40+ 家
- **市场**：ML (H2H), 让球 (spreads), 大小球 (totals), 冠军 (outrights)
- **接入**：REST API, JSON 格式, 支持 American/Decimal 赔率
- **对模型提升**：**极高** — 多源赔率可用 Wasserstein 距离做市场共识，历史赔率可训练赔率变化模式
- **注意**：不提供赛果/比分，需要与 ESPN（赛果）搭配使用

#### 暂不推荐：OddsPortal
- 无官方 API，纯网页爬虫，法律风险高
- 数据质量受反爬措施影响

### 3.2 球队/比赛数据

#### football-data.org
- **免费版**：12 个联赛，10 次/分钟，延迟赛果
- **€12/月 加强版**：实时比分，20 次/分钟
- **€29/月 Deep Data**：阵容+替补+进球者+红黄牌
- **€15/月 附加**：赔率附加 (40 联赛 HDA 赔率) 或 技术统计附加 (角球/控球/射门/犯规)
- **世界杯覆盖**：✅ 有 WC 联赛代码
- **代码现状**：`convert_football_data_to_espn_format()` 已实现，但转换后丢了 football-data 的丰富字段（仅保留赛果）
- **建议**：**不要无脑转 ESPN 格式**，直接利用 football-data 的原始结构

#### API-Football
- **免费版**：100 次/天（约 3-5 次完整赛程请求）
- **付费版**：~$25/月起
- **优势**：含 H2H 历史、球员统计、技术统计、场地数据
- **劣势**：免费版对世界杯预测完全不够用（100 次/天 = 3 次日更新）
- **代码现状**：`convert_api_football_to_espn_format()` 同样丢了原始字段

### 3.3 天气数据

#### OpenWeatherMap
- **免费版**：1,000 次/天（远超需求）
- **数据**：实时天气、5 天逐时预报、历史天气
- **对足球预测的影响**：
  - 雨/雪 → 进球数下降 15-25%（滑倒、球速变慢）
  - 高温 (>30°C) → 下半场体能下降，总进球减少
  - 强风 → 长传不准，防守方优势
- **接入**：`api.openweathermap.org/data/2.5/forecast?lat={lat}&lon={lon}&appid={key}`
- **成本**：免费
- **对模型提升**：**中等** — 极端天气下效果显著，但多数比赛天气中性

### 3.4 xG / 高级数据（⚠️ 重要更新）

#### FBref — 已不可用
- **2026 年 1 月**：Stats Perform (Opta) 终止与 FBref 的数据协议
- **原因**：Opta 成为 FIFA 官方博彩数据合作伙伴
- **现状**：FBref 已移除所有 xG/xA/射门等高级数据
- **结论**：❌ 不再推荐作为 xG 数据源

#### Understat
- **覆盖范围**：仅五大联赛（英超、西甲、德甲、意甲、法甲）
- **无世界杯数据** ❌
- **结论**：对世界杯预测无帮助

#### StatsBomb Open Data
- **免费**：GitHub 开源数据（JSON 格式）
- **包含世界杯**：✅ 2018 世界杯部分赛事
- **数据深度**：全场事件流 + xG（射门级）
- **用途**：离线训练模型、校准 λ 参数
- **结论**：✅ 推荐用于模型训练和校准，但非实时数据源

#### 当前最佳替代方案
| 需求 | 推荐方案 | 成本 |
|------|---------|------|
| 实时 xG | **无免费方案可用** → 转向 Opta 付费或 The Odds API 间接推断 | $30+/月 |
| 离线训练 xG | StatsBomb Open Data | 免费 |
| 球队实力评级 | Club Elo (免费) + FIFA 排名 | 免费 |

## 5. 实施建议

### 第一阶段（立即实施，零成本）
1. **The Odds API 免费版**：注册获取 API key -> 500 次/月足够世界杯单日使用
   - 接入多博彩公司 ML 赔率，做市场共识加权
   - 解决 ESPN ML 字段经常为空的问题
   - 接入方式：`GET https://api.the-odds-api.com/v4/sports/soccer_fifa_world_cup/odds/?apiKey={key}&regions=us,eu,uk&markets=h2h,spreads,totals`
2. **OpenWeatherMap 免费版**：1,000 次/天，接入球场天气预报
   - 只需在比赛日查询 8 个球场 x 3 时段 = 24 次/天
   - 数据作为 Poisson λ 的 soft 修正因子
3. **Club Elo 免费版**：替代 league_footprint 的 FIFA 排名分档
   - 提供更细粒度的球队实力评级

### 第二阶段（€12-30/月，中等投入）
4. **football-data.org 升级到 €12/月**：获得实时比分 + 稳定 API
   - 替代 ESPN 作为主要赛程/赛果数据源
   - 停止依赖不稳定的 ESPN 隐藏 API
   - 直接使用 football-data 原始 JSON 结构（不再转 ESPN 格式）
5. **The Odds API 升级到 $30/月**：获得历史赔率数据
   - 训练赔率变化模式识别
   - 关卡/淘汰赛阶段的赔率突变信号

### 第三阶段（可选项，低优先级）
6. **StatsBomb Open Data**：离线训练 Dixon-Coles ρ 参数
   - 用历史世界杯事件数据拟合更精确的 ρ
   - 不改变运行时数据流
7. **football-data.org Deep Data (€29/月)**：获取阵容+红黄牌数据
   - 增加关键球员缺阵检测
   - 对淘汰赛阶段有意义

### 架构建议

```
                     ┌───────────────────┐
                     │  Data Source Layer │
                     │                   │
  ESPN (免费) ───────┤ 赛程/赛果/状态    │
  football-data.org ─┤ 赛程/赛果/阵容    │
  The Odds API ──────┤ 多源赔率/历史赔率 │
  OpenWeatherMap ────┤ 天气数据           │
  Club Elo ──────────┤ 球队实力评级       │
                     └────────┬──────────┘
                              │
                              ▼
                     ┌───────────────────┐
                     │  Feature Pipeline │  ← 新增：数据融合层
                     │                   │
                     │ 赔率市场共识融合    │
                     │ 天气修正因子        │
                     │ 球队实力 Elo 值     │
                     │ xG 离线校准参数     │
                     └────────┬──────────┘
                              │
                              ▼
                     ┌───────────────────┐
                     │  Prediction Model  │  ← 现有，不修改
                     │  (Onside 4 + DC)  │
                     └───────────────────┘
```

**关键设计原则**：
- ESPN 仍保留作为免费 fallback（赛程/赛果基线）
- 新增数据源不修改现有 `calculate_prediction()` 签名
- 数据融合层产出标准化特征，模型只读特征不读原始数据
- 所有新数据源都有降级路径（数据源不可用 → 降级到 ESPN 纯赔率模式）
